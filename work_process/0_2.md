# 0_2. 마이크로컨트롤러(MCU)란?

> **"센서 → MCU → PC" 데이터 흐름 이해**

---

## MCU(Microcontroller Unit)란?

아주 작은 컴퓨터입니다.
CPU + 메모리 + 입출력 핀이 하나의 칩에 들어있어서
센서를 읽고, 계산하고, 다른 장치에 데이터를 보내는 역할을 합니다.

> 일반 컴퓨터와의 차이:
> - PC: OS 위에서 복잡한 프로그램 실행
> - MCU: 하나의 동작(센서 읽기, 모터 제어 등)을 반복 실행

---

## 아두이노(Arduino)란?

MCU를 쉽게 사용할 수 있도록 만든 **개발 보드 + 개발환경**입니다.
- MCU 칩 + USB 포트 + 핀 헤더가 한 보드에 있음
- Arduino IDE로 C-유사 언어로 코드 작성 → USB로 업로드 → 실행
- 초보자도 진입하기 쉬운 생태계 (라이브러리, 예제 코드 풍부)

---

## 이 프로젝트에서 MCU의 역할

```
[IMU 센서] ──UART/I2C──► [MCU] ──USB/BLE──► [PC]
[버튼]     ──디지털 핀──► [MCU]
```

- IMU 센서에서 데이터를 읽어들임
- 버튼 눌림 상태를 감지
- 이 데이터를 하나의 패킷으로 만들어 PC(또는 로봇팔 컨트롤러)로 전송

---

## WT61C-TTL 특수 사항

> **WT61C-TTL은 UART(TTL 시리얼) 출력이므로
> USB-to-TTL 어댑터를 쓰면 MCU 없이도 PC에 직접 연결됩니다.**

```
[WT61C-TTL] ──TTL 시리얼──► [USB-to-TTL 어댑터] ──USB──► [PC]
```

즉, Phase 1~2는 MCU(아두이노) 없이도 진행 가능합니다.
MCU는 **버튼 입력 추가(Phase 4)** 및 **무선화(Phase 6)** 단계에서 필요해집니다.

---

## MCU 선택지 (이후 단계에서 사용)

| 보드 | 특징 | 이 프로젝트 적합도 |
|------|------|------------------|
| Arduino Uno | 가장 기초, 입문용 | 버튼 테스트용으로 적합 |
| ESP32 | Wi-Fi + BLE 내장, 강력 | 무선화 단계에서 최적 |
| Raspberry Pi Pico | 저렴, Python 지원 | 중간 옵션 |

> **추천 경로:** 버튼 테스트는 Arduino Uno → 무선화 단계에서 ESP32로 전환
