# 미니 팔로우암 안정화 검증 (2026-06-11)

> 미니 팔로우암(=팔로워를 닮은 미니어처 조종용 팔, 리더암)으로 7축 텔레옵을 구현하고
> 통신·변환·안전·추종을 수치로 검증한 보고서.

## 0. 용어 정리

이 장치의 정확한 명칭은 **리더암(leader arm) / 마스터암**:
팔로워와 닮은 미니어처 팔을 **손잡이로 잡고 조종**하는 방식 (ALOHA / GELLO / SO-ARM 계열).
- "외골격(exoskeleton)"은 몸에 **착용**하는 형태를 가리키므로 부정확
- 컨트롤 분류: **joint-space retargeting** (리더 관절각 → 팔로워 관절각 1:1 복사),
  OpenArm ROS2 기준으로는 `forward_position_controller` 방식 + 자체 보간층

```
리더암 (사람이 손잡이로 움직임, 토크 OFF = 순수 엔코더)
  → 60Hz 폴링 (leader_arm.py / LeaderReaderThread)
  → 틱 차이 × 360/4096 × 부호 × 스케일 (+ 영점 오프셋)
  → 기존 안전 파이프라인 (동기화 보간 70°/s, 관절클램프, 충돌벽, freeze)
  → DYNAMIXEL 팔로워 7축
```

| 서보 | **Feetech STS3215 계열** (다이나믹셀 아님) | P2.0 무응답 + P1.0 핑 응답, addr3=777 시그니처 |

| 현재위치 레지스터 | **addr 56** (2B, 0~4095) | addr36은 상수(2640)=위치 아님, addr56만 ID별 상이·정지시 안정 |

## 3. IMU 웨어러블 vs 리더암 비교

| | IMU 웨어러블 | 리더암 |
|---|---|---|
| 측정 원리 | 자세(쿼터니언) → 보정축 분해 | 관절각 **직접 측정** (엔코더) |
| 보정 | functional 보정 + 2-포즈 재정렬 필요 | **불필요** (시작 시 기준 캡처만) |
| 짐벌락/드리프트 | 쿼터니언으로 해결했으나 지자기 환경 영향 | **원천 없음** |
| 정확도 | 보정축 오차 수° (어깨 8° 언더리드 잔존) | 0.088° 분해능, 축오차 개념 없음 |
| 갱신율 | ~30Hz (펌웨어 115200 후) | 60Hz (최대 400Hz) |
| 지연 | ~100-150ms | **~60ms** |
| 장점 | 장비 없이 맨팔 동작, 야외/이동 자유 | 정밀·즉답·무보정 |
| 약점 | 보정/장착 변동/센서융합 지연 | 장치 앞에 있어야 함 |

→ **리더암 = 정밀 기준기(ground truth)**: 같은 동작을 IMU와 리더로 비교하면 IMU 보정 오차를 정량 측정 가능.

## 4. 안전 (두 모드 공통 파이프라인 공유)

- 동기화 보간(경로모양 보존, 70°/s) · 관절클램프(±90~140°) · 팔꿈치 충돌벽(EEPROM 800틱)
- 리더 전용: 끊김 0.3s freeze → 복구 램프인 / **리더 오류 시 자동 전달중지** / 토크 OFF 강제(엔코더 모드)
- 전달 시작 시 모터5가 +90°로 부드럽게 이동(영점 정렬) — 정상 동작

## 핵심 발견 2건

1. **파이프라인 무죄 증명**: 보정축만 정확하면 어깨/팔꿈치급 정확도(0.9~10°)가 나온다.
   사람 착용 시 오차의 주범은 보정/장착이지 소프트웨어가 아님.
2. **손목 가지점프 재현·규명** ⭐: j5가 0-30s 완벽(오프셋 0.0°) →
   **j6(손목피치) 대각도 통과 순간 +90° 가지로 점프 후 고착** (오프셋 추이로 시점 특정).

   = 기존 미해결 "ID7 한번씩 이상" 증상의 정체. 운용범위(±90 클램프) 안에서도
   특이 근처 통과 시 발생 가능 → **가지 연속성 스냅**(이전 출력에 가장 가까운 동등해 선택,
   이산 선택만이라 감김 불가) 또는 손목피치 운용제한이 다음 개선 후보.
