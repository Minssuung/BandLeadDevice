# IMU 텔레옵 정확도 검증·개선 보고 (2026-06-11)

## 어떻게 검증했나 — IMU 센서를 리더암에 직접 부착 ⭐

이번 검증의 핵심 아이디어:

> **IMU 센서 3개를 리더암(미니 팔로우암)의 윗팔/아랫팔/손 부분에 직접 붙였다.**
> 리더암의 각 관절에는 정밀 각도 센서(0.088° 단위)가 들어 있다.
> 그래서 리더암을 움직이면 — **"진짜 각도"(리더암 센서)와 "IMU가 계산한 각도"를
> 같은 움직임에 대해 직접 비교**할 수 있다.

```
리더암을 손으로 움직임
   ├─ 리더암 관절 센서 → 진짜 각도 (정답지)
   └─ 붙여놓은 IMU 3개 → 우리 시스템이 계산한 각도
            ↓
        둘을 비교 = IMU 시스템의 오차가 그대로 보임
```

지금까지는 "느낌상 덜 올라가는데?" 수준이었다면, 이제는 **관절마다 몇 도 틀리는지 숫자로** 나옵니다.
사람 팔에 차고 비교하면 사람 몸의 출렁임이 섞이는데, 리더암은 단단한 기계라 순수하게
IMU와 계산만 평가됩니다.

만든 도구:
- 동시 기록 도구 (IMU값 + 리더암 정답을 같은 시간에 저장)
- **자동 보정 도구**: 리더암 정답을 보고 IMU의 축 방향을 자동으로 찾아줌
  (가짜 데이터로 시험: 숨겨놓은 축을 완벽히 찾아냄, 오차 0.00°)
- 드리프트 측정 도구 (사람이 차고도 사용 가능 — 같은 자세로 돌아왔을 때 얼마나 틀어졌나)

---

## 1. 각도 계산(분해) 개선 — 오차 8.8% → 0%

IMU가 잰 "자세"를 "관절 각도들"로 쪼개는 계산에 두 가지 문제가 있었고, 모두 해결:

| 문제 | 쉽게 말하면 | 해결 | 결과 |
|------|------------|------|------|
| 큰 손목 회전에서 계산 실패 (8.8%) | 안전장치(출력 제한 범위)가 너무 좁아서 진짜 답이 범위 밖에 있으면 못 찾음 | 여러 출발점에서 풀어보고, **"검산"(답을 다시 합성해 원본과 비교)을 통과한 답만 채택** | 실패율 **0%** |
| 특정 자세 지나면 각도가 갑자기 90° 점프 | 같은 자세를 표현하는 답이 여러 개라서, 지나가다 다른 답으로 갈아탐 | **직전 값과 가장 가까운 답을 선택** (점프 방지) + 한 프레임짜리 점프는 모터로 안 보냄 | 점프 차단 |

그리고 중요한 확인: **천천히 움직일 때 계산 오차 = 0.0°.**
→ 소프트웨어는 이제 오차를 만들지 않습니다. 남은 오차는 전부 센서/통신 몫.

## 2. 센서 모드 결정 — "손목이 가끔 이상해지는" 미스터리 종결 ⭐

오랫동안 "손목(ID7 등)이 한 번씩 이상하게 돈다"는 증상이 있었는데, 리더암 검증으로 범인을 잡았습니다:

**범인 = 지자기(나침반) 간섭.** IMU는 방위를 나침반으로 잡는데, 모터 자석·철제 책상 옆에서는
나침반이 흔들립니다. 센서마다 다르게 흔들리면 그 차이가 **손목 회전값에 통째로 들어가서**
어느 순간 90~100°씩 틀어졌던 겁니다. (리더암 실험에서 그대로 재현 + 시점까지 특정)

나침반을 끈 모드(6축)와 켠 모드(9축)를 **사람이 직접 차고 비교**했습니다:
같은 자세로 돌아왔을 때 얼마나 틀어져 있는지를 반복 측정 —

| | 나침반 OFF (6축) | 나침반 ON (9축) |
|---|---|---|
| 윗팔 | 최대 **166°** 틀어짐 | **13°** |
| 팔꿈치 | 133° | **4°** |
| 손목 | 124° (계속 누적됨) | **22°** (누적 안 됨!) |

**결론: 사람이 차는 밴드는 나침반 ON(9축)이 정답.**
몸은 모터 자석에서 충분히 멀어서 나침반이 제 역할을 하고, 틀어짐이 누적되지 않습니다.
(반대로 리더암에 붙여 검증할 땐 자석이 바로 옆이라 나침반 OFF가 유리 — 상황별로 스위치 하나로 전환 가능하게 만들어 둠)

## 3. 통신 개선 — 한 줄 고치고 3배 빨라짐

| | 전 | 후 |
|---|---|---|
| 초당 데이터 수신 | 9~14회 (널뛰기) | **초당 30.8회 안정** |
| 반응 지연 | 약 0.13초 | **0.04초 이하** |
| 통신 실패 횟수 | 수천 건 누적 | **0건** |

원인이 허무한데 — 펌웨어가 디버그용으로 **데이터 전문을 매번 시리얼에 출력**하고 있었고,
그 출력 한 번에 35ms씩 전체가 멈춰 있었습니다. 출력을 2초에 한 번으로 줄이자
하루 종일 따라다니던 "느림, 끊김, 통신실패"가 전부 사라졌습니다.

**개선 효과 (실측)**: 어깨를 237° 크게 계속 휘둘러도 추적 오차 **평균 3.1°** —
개선 전(14~15°)의 약 1/5.

## 4. 안전장치 추가

| 장치 | 하는 일 |
|------|--------|
| 팔꿈치 충돌벽 | 모터 자체에 "이 각도 밑으로 절대 금지"를 저장 (앱이 꺼져도, 뭘 보내도 프레임에 못 부딪힘) |
| 위치 급변 차단 | 시작 위치 계산이 틀어져도 모터가 휙 돌지 못하게 (항상 제한 속도로만 이동) |
| 점프 차단 | 계산값이 한 순간 크게 튀면 일단 보류, 3번 연속 같은 값일 때만 따라감 |
| 드리프트 감시 | 관절값이 몰래 조금씩 흘러가면 "기준 다시 잡으세요" 경고 |
| 센서 리셋 감지 | 센서가 재부팅되면 즉시 경고 |

안전장치가 갖춰진 덕분에 **모터 속도도 82°/s → 206°/s로 해방** — 사람 팔 속도를 거의 그대로 따라옵니다.
