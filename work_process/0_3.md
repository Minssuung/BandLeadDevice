# 0_3. I2C 통신이란?

> **IMU를 MCU에 연결하는 방법 — 핀 2개로 데이터 주고받기**

---

## I2C란?

**Inter-Integrated Circuit**의 약자로, 여러 장치(센서, 디스플레이 등)를
단 **2개의 신호선**으로 연결해서 데이터를 주고받는 통신 방식입니다.

```
[MCU] ──SDA──► [센서 1]
      ──SCL──► [센서 1]
            ──[센서 2]
            ──[센서 3]  ← 같은 선에 여러 장치 연결 가능
```

| 핀 이름 | 역할 |
|--------|------|
| SDA (Serial Data) | 데이터를 주고받는 선 |
| SCL (Serial Clock) | 타이밍을 맞추는 클럭 선 |

---

## UART(TTL 시리얼)와의 차이

> **WT61C-TTL은 I2C가 아닌 UART 방식을 사용합니다.**

| 방식 | 핀 수 | 특징 | WT61C-TTL |
|------|-------|------|----------|
| UART (TTL 시리얼) | TX, RX, VCC, GND (4핀) | 1:1 연결, USB-to-TTL로 PC 직결 가능 | **이 방식 사용** |
| I2C | SDA, SCL, VCC, GND (4핀) | 1:다 연결, 같은 선에 여러 장치 | 일부 IMU 모듈에서 사용 |
| SPI | MOSI, MISO, SCK, CS (4~5핀) | 고속, 1:1 또는 칩 선택 방식 | 고급 센서에서 사용 |

---

## I2C 주소 충돌 문제

I2C를 쓸 경우, 같은 모델의 센서를 2개 이상 연결하면 주소가 겹쳐서 충돌합니다.
WT61C-TTL은 **UART 방식**이라 이 문제가 없습니다.
→ 센서마다 별도 USB-to-TTL 어댑터를 쓰면 독립 포트로 인식됩니다.

---

## 이 프로젝트에서 I2C가 필요한 시점

향후 PCB나 최종 하드웨어를 설계할 때 IMU를 MCU에 직접 연결한다면
I2C 또는 SPI 방식이 필요해질 수 있습니다.
하지만 **프로토타입 단계(Phase 1~5)에서는 UART + USB 방식으로 충분**하며,
I2C는 Phase 6(하드웨어 통합) 이후에 고려하면 됩니다.

---

## 정리

```
지금 단계에서 알아야 할 것:
  WT61C-TTL → UART 출력 → USB-to-TTL 어댑터 → PC USB
  (I2C 지식은 아직 몰라도 됨)

나중에 필요할 것:
  MCU + IMU 직접 연결 설계 시 I2C 또는 SPI 공부
```
