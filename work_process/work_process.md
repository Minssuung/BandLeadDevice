## 최종 목표 시스템 구성

| 부위 | 센서 | 제어 자유도 |
|------|------|-------------|
| 윗팔 (이두근 밴드) | IMU × 1 | 3 DOF (어깨 Roll/Pitch/Yaw) |
| 아랫팔 (전완근 밴드) | IMU × 1 | 1 DOF (팔꿈치 Pitch) |
| 손잡이 | IMU × 1 | 3 DOF (손목 Roll/Pitch/Yaw) |
| 손잡이 버튼 | 버튼 × 2 | 1 DOF (그리퍼 집기/펴기) |
| **합계** | | **7 DOF + 그리퍼 1 DOF** |

## 현재 구현 상태 (2026-05 기준, 실측)

> ⚠️ **중요:** 아래 Phase 로드맵은 초기 계획이며, 실제 구현은 하드웨어 방식이 바뀌었습니다.
> 이 섹션이 **현재 실제 동작하는 시스템**의 기준입니다.

### 초기 계획 → 실제 구현 변경점

| 항목 | 초기 계획 (아래 Phase 본문) | **실제 구현 (현재)** |
|------|---------------------------|---------------------|
| IMU 모델 | WT61C-TTL (UART) | **WT901C-485 (RS485 Modbus RTU)** |
| IMU 연결 | USB-to-TTL 어댑터로 PC 직결 (유선) | **ESP32(LOLIN D32) + MAX485 → Wi-Fi UDP (무선)** |
| 모터 보드 | OpenRB-150 | DYNAMIXEL 직결 (USB, `/dev/ttyACM0`) |
| 규모 | IMU 1개 → 모터 3축 | **IMU 3개 → 모터 7축** |
| 로봇팔 | OpenArm (ROS2) | **아직 미연결** (DYNAMIXEL 7축 테스트 리그로 대체 중) |

### 실제 데이터 흐름

```
IMU×3 (WT901C-485) ─RS485 Modbus─► ESP32 ─Wi-Fi UDP(:4210)─► PC GUI ─USB─► DYNAMIXEL ×7
                                  (5V 보조배터리 전원)        (192.168.0.x)   (/dev/ttyACM0)
```

## 미해결 이슈 & 다음 할 일

> 출처: [Confluence — IMU 3개·7DoF 매핑 테스트](https://pinkwink.atlassian.net/wiki/spaces/CR/pages/3439951878/IMU+3+-+7DoF) (2026-05-29)
> 현재 "IMU→ESP32→PC→모터 7축"까지는 동작 확인됨. 아래 3가지가 해결 과제.

### 이슈 1. 정확도 부족
- 증상: 이동 방향 오차, 관절 회전량 차이, 특정 자세에서 과도한 움직임, 부자연스러운 관절 반응
- 대응: **캘리브레이션 기능** + 관절별 제한 범위(클리핑) + 축 부호(±) 실측 보정

### 이슈 2. 통신 지연 (Latency)
- 증상: 실제 팔보다 로봇팔 반응이 약간 늦음
- 의심 구간: IMU→ESP32 / ESP32→PC / PC→모터
- 대응: 데이터 처리 최적화, 통신 속도 개선, 폴링 주기/패킷 구조 점검

### 이슈 3. 비정상 동작 — 짐벌락 (Gimbal Lock)
- 증상: 임계값 이상 입력 시 로봇팔이 갑자기 엉뚱한 방향으로 크게 튐
- 원인: **Euler Angle 기반 처리의 짐벌락**
- 대응: Quaternion 기반 자세 처리로 전환, 이상값 제거 로직, 움직임 스무딩

### 개선 방향 (Confluence 4.3)
1. **캘리브레이션 도입** — 사용자 팔 동작 ↔ 로봇 동작을 2~3회 측정해 보정값 생성 → **JSON 저장/로드**
   (착용 시마다 생기는 장치 위치 편차도 보정)
2. **SW 개선** — 센서 필터링 · 관절 제한 범위 · 이상값 제거 · 통신 최적화 · 스무딩 · 자세 변환 알고리즘 개선

### Phase 1. IMU 센서 단품 실험 — "센서가 뭘 출력하는지 눈으로 확인"

> 보유 센서: **WitMotion WT61C-TTL**
> TTL 시리얼(UART) 출력 방식 → USB-to-TTL 어댑터로 PC에 직접 연결 가능.
> Arduino 없이도 PC에 바로 꽂아서 데이터를 볼 수 있습니다.

#### 1-1. 재료 준비
- [O] IMU 센서: WitMotion WT61C-TTL (보유 완료)
- [O] **USB-to-TTL 어댑터** 구매 (CH340 또는 CP2102 기반 동글, 수백 원~수천 원)
  - WT61C-TTL의 TX/RX/VCC/GND 4핀을 이 어댑터에 연결하면 PC에서 COM 포트로 보임
  - 이미 있거나 센서 패키지에 포함된 경우 생략
- [O] 점퍼 와이어 4개 (암-수 또는 암-암, WT61C 핀 형태에 맞게)

#### 1-2. 연결 방법 (납땜 없음)

```
WT61C-TTL 핀  →  USB-to-TTL 어댑터 핀
  VCC (5V)   →  5V (또는 3.3V, 모듈에 따라 다름)
  GND        →  GND
  TX         →  RX
  RX         →  TX  ← TX/RX 교차 연결에 주의!
```

- [O] 위 연결 후 USB-to-TTL 어댑터를 PC USB에 꽂기
- [O] 장치 관리자(Windows) 또는 `ls /dev/ttyUSB*`(Linux)에서 COM 포트 확인

#### 1-3. WitMotion 전용 소프트웨어로 즉시 시각화 (가장 쉬운 방법)
- [X] WitMotion 공식 소프트웨어 다운로드
  - Windows: [WitMotion 3D 데모 소프트웨어](https://www.wit-motion.com/download.html)
  - 소프트웨어 실행 → COM 포트 선택 → baud rate **9600** → 연결
- [X] 센서를 기울이면 화면 3D 모델이 따라 움직이는지 확인
- [X] **목표 확인:** Roll, Pitch, Yaw 값이 실시간으로 변하는 것을 눈으로 확인

### 1-3-2. 1_IMU_sensor_test 로 대체
- [O] 1_IMU_sensor_test 폴더에 imu_sensor_test.py 파일 실행
  - usb 목록에서 연결된 usb 확인후 select box에서 클릭,
  - baud 를 115200으로 맞추기
  - 연결 버튼 클릭

#### 1-4. 데이터 확인 및 검증
- [O] 센서를 평평하게 놓았을 때의 기준값(Roll=0, Pitch=0, Yaw=0 근처) 확인
- [O] 센서를 앞으로 기울이면 → **Pitch** 값 변화 확인
- [O] 센서를 옆으로 기울이면 → **Roll** 값 변화 확인
- [O] 센서를 수평 회전시키면 → **Yaw** 값 변화 확인
- [O] 가속도(Ax, Ay, Az), 자이로(Wx, Wy, Wz) 원시 데이터도 확인
- [O] **결론:** WT61C-TTL은 Euler 각도를 내부 계산해서 바로 출력함 → 별도 센서 퓨전 알고리즘 불필요

### Phase 2. IMU 1개가 모터 3개에 명령을 줄 수 있는지 실험

> 보유 모터: **DYNAMIXEL XL330-M288-T** (TTL 통신, Protocol 2.0, 0~4095 position / 1회전)
> 보유 보드: **ROBOTIS OpenRB-150** (USB-C로 PC 연결, DYNAMIXEL TTL 포트 제어 가능)
> IMU의 Roll/Pitch/Yaw 3축 값을 모터 3개에 각각 1:1로 대응시켜 실시간 제어가 되는지 확인합니다.

#### 2-1. 재료 준비
- [O] DYNAMIXEL XL330-M288-T 3개 준비
- [O] OpenRB-150 1개 준비
- [O] XL330과 OpenRB-150를 연결할 수 있는 DYNAMIXEL 3핀 케이블 준비
- [O] 모터 전원 방식 확인
  - OpenRB-150의 DYNAMIXEL 전원 설정을 확인하고, 필요하면 XL330 정격에 맞는 외부 전원 사용
  - 모터 3개를 동시에 움직일 때는 USB 전원만으로 부족할 수 있으므로 무부하 상태부터 시험

#### 2-2. 연결 방법
- [O] OpenRB-150를 USB-C로 PC에 연결
- [O] OpenRB-150의 DYNAMIXEL TTL 포트에 XL330 3개를 체인 방식으로 연결
- [O] 각 모터의 ID를 서로 다르게 설정
  - 예: ID 1 = Roll, ID 2 = Pitch, ID 3 = Yaw
- [O] 각 모터의 baud rate와 Protocol 2.0 설정 확인

#### 2-3. 기본 통신 확인
- [O] **DYNAMIXEL Wizard 2.0**으로 OpenRB-150에 연결된 모터가 모두 인식되는지 확인
- [O] 각 모터의 현재 위치를 읽고, 소량의 목표 위치 명령을 보내 정상 응답하는지 확인
- [O] 중앙 위치를 기준 자세로 정함
  - 예: 2048을 중립값으로 두고 좌우 회전 범위를 제한해서 테스트

#### 2-4. Python 제어 실험
- [O] Python에서 IMU 값을 읽는 기존 코드와 별도로 DYNAMIXEL 제어 테스트 코드 작성
- [O] **DYNAMIXEL SDK (Python)** 설치
- [O] Roll/Pitch/Yaw 값을 각 모터의 Goal Position으로 변환
  - 각 축별로 부호가 다를 수 있으므로 방향 반전 옵션 준비
  - 급격한 점프를 막기 위해 최소/최대 위치 제한과 속도 제한 적용
- [O] 처음에는 한 축만 연결해서 검증한 뒤, 3축 동시 제어로 확장

#### 2-5. 목표 확인
- [O] IMU를 기울이면 대응되는 모터가 같은 방향으로 따라 움직이는지 확인
- [O] 3개 모터가 동시에 갱신될 때 끊김 없이 반응하는지 확인
- [O] **결론:** WT61C-TTL의 Euler 각도를 OpenRB-150 + XL330 조합으로 실시간 위치 명령에 매핑 가능함을 확인
