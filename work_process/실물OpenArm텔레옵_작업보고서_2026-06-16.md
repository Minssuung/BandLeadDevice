## 0. 요약

미니 팔로우암(DYNAMIXEL)에서 검증한 IMU 텔레옵을 **실물 OpenArm v1**(caring_openarm 워크스페이스)으로 이식했다. ROS2 Jazzy 환경을 구축하고, 실물 CAN/모터를 연결해 기본 동작(관절 제어)을 확인했으며, **BandLead 텔레옵 어댑터**(IMU→JTC)와 **앱 텔레옵 탭**을 구현했다. fact-check로 안전 결함을 잡아 충돌 보호·E-stop을 보강했다.

## 1. 목표 & 시스템 구성

```
[IMU 밴드 ×3] ─RS485─▶ [ESP32] ─UDP 4210─▶ [PC: 텔레옵 어댑터]
  (어깨/상완/전완)                          calib_ik 분해 → 7관절각
                                            부호/매핑 + 안전장치
                                                    │ ROS 토픽
                                                    ▼
                            [bringup: ros2_control + JTC + HW(CAN-FD)]
                                                    ▼
                                    [실물 OpenArm v1 우완 7-DOF + DAMIAO]
```

- **caring_openarm**: 실물 OpenArm v2 양팔 제어 colcon 워크스페이스(ROS2 Jazzy+MoveIt2+ros2_control). 실물은 **v1** 사용.
- **BandLead**: IMU 밴드(WT901C 9축) → ESP32 RS485 Modbus → UDP. calib_ik 멀티스타트 분해.

## 4. 기본 동작 검증 (관절 제어 + 부호/매핑)

- **관절 제어 탭**: 슬라이더 → MoveIt 충돌회피 계획 → JTC 실행 → **실물 관절 이동 확인**(앱이 oa_movegroup_v1_real 자동 기동, controller-free라 bringup과 비충돌).
- **부호/매핑 (viz→실물 검증)**:
  - 우완 **j1(yaw)·j2(pitch) 부호** — 양팔 axis가 동일(미러 누락)이라 좌우 부호 반대 필요. j1은 userSign[-1]만으로 맞음(FLIP 시 이중반전 클램프), j2는 실물에서 부호 반대 발견(미해결).
  - **j6↔j7 스왑**: viz는 v2(j6=Y·j7=X), **v1은 축 반대(j6=X·j7=Y) → 스왑 OFF**가 맞음. signmap 기본값을 v1용으로 생성.
  - j2 가동범위 비대칭(우완 -10°~+190°): 안쪽 끝은 자기충돌 영역.

## 5. BandLead 텔레옵 어댑터 (`openarm_teleop_adapter.py`)

IMU UDP(4210) → `bridge`(calib_ik 분해 재사용) → 부호/매핑 → `/{side}_joint_trajectory_controller/joint_trajectory` **직접 발행**(MoveIt 우회 = 저지연).

| 기능 | 내용 |
|---|---|
| 분해 | bridge.calib_ik(멀티스타트·브랜치연속성·FK잔차게이트) 재사용 |
| 부호/매핑 | signmap JSON(`~/.local/share/openarm_monitor/teleop_signmap.json`) **실시간 reload**(10패킷마다 mtime) |
| 안전 | **DRY-RUN 기본**·stale-freeze(200ms)·속도클램프 180°/s·점프디바운스·관절한계 클램프 |
| 기준 | 차렷 자동기준(첫 패킷) + **SIGUSR1 즉시 재설정**(재시작 없음) |
| 시뮬 뷰어 | `--viewer` → MuJoCo 창(목표 자세 + 충돌점 빨강), DRY-RUN 미리보기 |

## 7. 관절 한계·충돌 방지 — OpenArm 표준 구조 & 어댑터가 자체 구현한 이유

### 7.1 OpenArm 표준(caring_openarm)은 어느 계층에서 하나
**관절값 제한:**
| 계층 | 강제? | 내용 |
|---|---|---|
| URDF (`v1_tcp.urdf` ← `joint_limits.yaml`) | ❌ 정의만 | `<limit>` lower/upper/effort/velocity. right_j1 -80~200°·j2 -10~190° 등 — 메타데이터 |
| MoveIt 계획 (OMPL) | ✅ 계획 시 | URDF 한계 + 속도/가속(`joint_limits.yaml`, 기본 0.1배) 존중한 궤적만 생성 |
| ros2_control JTC | ❌ | 계획 궤적 그대로 추종 — 위치 한계 클램프 **안 함**(`oa_ros2_controllers.yaml`은 추적오차 constraints만) |
| HW(`openarm_simple_hardware.cpp`) | ❌ | `write()`가 `pos_commands_` 클램프 안 함(게인 kp/kd만 `std::clamp`) |

**모터/프레임 충돌 방지:**
| 계층 | 방식 |
|---|---|
| SRDF (`openarm_bimanual.srdf`) | `disable_collisions` **28쌍**(인접 링크·그리퍼 손가락쌍) 제외, 나머지(양팔간·팔↔베이스 body_link0)는 검사 |
| MoveIt 계획 | OMPL이 SRDF 기반 self/환경 충돌 검사 → 충돌 경로 기각(계획만, 실행 중 미검사) |
| MoveIt Servo (`oa_servo_v1.launch.py`) | 실시간 **10Hz**, `self_collision_proximity 0.01m`·`scene 0.02m` 근접 시 정지 — **servo 실행 시만** |
| JTC / HW | ❌ 충돌 방지 **없음** |
| 그리퍼↔프레임 | SRDF에 손가락-프레임 쌍 **미정의** → planning_scene 외부 장애물로 따로 모델링해야 함 |

### 7.2 ⭐ 직접 JTC 발행(MoveIt 우회) 시 — 표준 보호가 전부 빠짐
OpenArm 표준의 관절 한계·충돌 방지는 **전부 MoveIt 계획/Servo 계층에만** 존재한다. 그런데 **BandLead 어댑터는 저지연을 위해 `/{side}_joint_trajectory_controller/joint_trajectory`에 직접 발행(MoveIt 우회)** 한다 → **URDF 한계도, SRDF/MoveIt 충돌 검사도 전혀 적용되지 않는다.**

→ 따라서 어댑터에 **자체적으로** ①URDF 관절한계 클램프 ②MuJoCo 실시간 충돌 검사를 넣은 것이 **필수**였다(아래 §8). 특히 그리퍼-프레임은 SRDF에도 미정의라 표준만으로는 절대 못 막는다.

## 8. 안전 — 어댑터 자체 구현 (fact-check & 충돌 보호 P0)

위 §7.2대로 텔레옵은 MoveIt 표준 보호를 우회하므로 어댑터에 직접 구현. **fact-check가 결함을 잡았다:**

| fact-check 발견 | 수정(P0) |
|---|---|
| 충돌검사 `ncon>base`가 그리퍼 손가락쌍 접촉에 흔들려 **검출 누락** | base 폐기 → **우완 collision 연루 접촉만 + 손가락쌍 제외 + 거리 판정** |
| **fail-open**(scene 없으면 무방비 publish) | **fail-closed**(충돌검사 불가 시 EXECUTE 거부) |
| E-stop이 **0.3s UI 폴링** 의존(브라우저 의존) | `source.estop()`→플래그 파일→**어댑터 직접 폴링**(매 패킷) |
| freeze가 "되돌아가기" 위험 | 충돌 시 **현재 실측 자세(/joint_states) 홀드** |

**그리퍼-프레임 충돌**: 모델 collision이 실물보다 **~5cm 큼**(실물 차렷 5cm 여유인데 모델 0mm) → **오프셋 보정**(실물거리=모델거리+5cm, 실물 1cm에서 차단). 정밀 검증은 시뮬 뷰어로 닿는 자세 확인 중.

### 8.1 실시간 성능 — 충돌 검사가 텔레옵을 늦추지 않는가
**측정 결과 충분히 빠름.** 충돌 검사 1회 = MuJoCo `mj_forward` **0.082ms**(≈12kHz 처리 여력). IMU 패킷은 30~50Hz(20~33ms 간격)로 들어오므로, 검사가 명령보다 **약 250배 빠르다** → 검사를 매 패킷 돌려도 텔레옵 추종에 지장 없음.

**빠른 이유:** 중력·관성으로 팔이 떨어지고 흔들리는 것까지 적분하는 **동역학 시뮬(`mj_step`)이 아니라**, "이 자세에서 어디가 닿나"만 보는 **자세 계산 + 접촉 검사(`mj_forward`)만** 수행하기 때문. 즉 물리 시뮬이지만 가볍다.

| | 시간 |
|---|---|
| 충돌 검사 1회 | **0.082ms** |
| IMU 명령 간격 | 20~33ms (30~50Hz) |
| 여유 | 약 250배 (검사 후 ~19.9ms 남음) |

→ 실측: DRY-RUN 시뮬 뷰어도 충돌 검사를 켠 채 끊김 없이 IMU를 추종(목표OA 실시간 갱신).
