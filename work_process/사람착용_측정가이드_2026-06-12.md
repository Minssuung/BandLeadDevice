# 사람 착용 텔레옵 검증 — 단일 프레임워크 측정 가이드 (2026-06-12)

IMU 밴드를 **사람 몸에 장착**한 실사용 조건 전용. 지그(리더암 부착) 안 함.
모든 측정을 **하나의 프레임워크**(앱 내장 검사 기록 + inspection_report.py)로 통합.

## 측정으로 얻는 것 (사람 착용 = '반증' 성격)

| KPI | 측정 | 도구 | 성격 |
|---|---|---|---|
| K4 정지 배회 | ✅ | 앱 검사기록 + report | 사람 떨림 포함 상한치 |
| K5 드리프트 | ✅ | 앱 검사기록 + report (drift_ab_test 흡수) | 추세 회귀 |
| K1 종단 지연 | ✅ | 슬로모 240fps (영상 필요, 별도) | ~110ms 실측 |
| C→P 출력단 | ✅ 참고 | report 자동 | 합부 무관 |
| K2 추종/K3 게인/K6 반복 | ❌ | — | GT(리더) 없어 불가 |

> **원칙**: 사람 착용 합격은 "정확도 인증"이 아니라 "결함 없음(반증)". 불합격이 나오면 그건 진짜 결함.

## 측정 절차 (HM 시나리오 = 한 번의 기록으로 K4+K5)

### 준비
1. IMU 밴드 3개 착용(윗팔/아래팔/손), 9축 모드(현 펌웨어 기본), 전원 인가 후 2~5초 정지(자이로 자동캘)
2. 앱 실행 → 모터 연결 → IMU 연결 → [기준 설정](차렷 자세)
3. 검사 입력칸에 `HM_human` 입력

### 기록
4. **[검사 기록 시작]** (이때부터 **Space 키 = 정지 마크** 활성)
5. 다음을 **5~6회 반복**:
   - **중립(차렷) 자세로 3초 정지** → 정지 시작에 **Space**, 3초 후 정지 끝에 **Space**
     - ⚠ **마우스 클릭 금지** — IMU가 흔들려 정지구간 오염. 반드시 **키보드 Space**로.
     - Space는 짝수 번째마다 자동으로 시작→끝 토글 (로그에 "정지 시작/정지 끝" 표시)
   - 전 관절 크게 휘젓기 20초 (어깨 벌림/굽힘, 팔꿈치, 손목 롤·굴곡 골고루)
6. 마지막 중립 정지 + Space 쌍으로 마무리
7. **[검사 기록 중지]** (Space 일반 동작 복귀)

→ **드리프트(K5) 산출 요건: 정지구간 4개 이상 + 첫~끝 정지 시간 span 60초 이상.**
   5~6회 반복(총 ~2분)이면 충족. 배회(K4)는 정지구간 1개부터 산출.

### 분석
```bash
cd ~/dev_ws/BandLeadDevice/work_process/esp_imu_pc_control
python3 inspection_report.py inspection_logs/<HM파일>.csv
```
→ K4 배회 + K5 드리프트 + C→P 출력단 참고표 자동 산출, 등급 판정, 보고서+그래프.

### 종단 지연(K1)은 슬로모로 (선택)
- 폰 240fps로 사람 팔 + 로봇을 한 화면에 + 반전 동작 5회 → 이미 ~110ms 실측됨.
- 분석: `imu_gap_probe.py` 동류의 상호상관(지연개선_2026-06-12.md 참조).

## 판정 기준 (inspection_criteria.json, B등급=실사용 목표)

| KPI | A | B | C |
|---|---|---|---|
| 배회 wander_deg | 0.5 | 1.5 | 3.0 |
| 드리프트 drift_dpm (기본) | 0.5 | 2.0 | 5.0 |
| 드리프트 drift_dpm (j5 손목롤) | 2.0 | 10.0 | 30.0 |

> j5(손목롤=전완축 회전)는 IMU 상대 yaw 드리프트의 구조적 희생자라 기준 완화. 여기서 불합격 나오면 알려진 약점 확인.

## 주의

- **드리프트는 정지구간 4개+ & span 60s+**에서만 산출(미달이면 미측정→판정불가). 5~6회 반복이면 충분.
  - 정지구간이 앞쪽에 몰리면(span<60s) 추세 외삽이 불안정해 산출 안 함 — 일부러 거부(FC 권고).
- Space 토글이라 짝수로 눌러야 쌍 완성. 홀수면 마지막 1개는 무시됨.
- 안전정지(K7)는 이 프로토콜에서 제외(사용자 결정). stale freeze 기능 자체는 코드에 유지.
- 측정기 신뢰성: 보고서 머리말 "자기검증 34/34 통과"가 매번 자동 기록.
  - FC 검증으로 잡은 함정 반영됨: ±180° 랩어라운드(j5류), 군집 외삽, 동작구간 오인, 배회/드리프트 이중계상.
