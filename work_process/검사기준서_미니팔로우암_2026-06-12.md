# 미니 팔로우암 텔레옵 장치 검사 기준서 (2026-06-12, v1.1 — FC 검증 반영)

> 본 문서는 **검사원 관점의 블랙박스 검사 기준**이다. 장치 내부 구현(분해 알고리즘, 안전 로직 등)을 일절 가정하지 않고, 장치 외부에서 측정 가능한 **입력(조작 동작) 대비 출력(팔로워암 동작)**만으로 성능을 정량 판정한다.
> 내부 구현 검증(결함 주입, 회귀)은 별도 문서 `미니암_테스트플랜_OpenArm전환게이트_2026-06-12.md`(엔지니어링 시험)로 분리한다.

## 1. 검사 대상과 범위

| 항목 | 내용 |
|---|---|
| 피검 장치 | IMU 밴드(3개) / 리더 외골격 → 7축 팔로워암(DYNAMIXEL) 텔레오퍼레이션 시스템 일체 |
| 검사 경계 | 입력: 조작자의 관절 동작(기준기로 계측) / 출력: 팔로워암 관절각(모터 엔코더) |
| 제외 | 내부 신호(파이프라인 목표각 등)는 판정에 사용하지 않음(원인 분석 참고만) |

## 2. 측정 구성 (기준기와 추적성)

- **기준기(ground truth) = 리더 외골격** (Feetech STS, 12-bit 엔코더, 분해능 0.088°). 두 용도:
  - **리더 모드 검사**: 리더가 입력 장치이자 기준기.
  - **IMU 모드 검사(지그)**: IMU 밴드를 리더암 링크에 결속 → 같은 동작을 IMU(피검 경로)와 리더 엔코더(기준 경로)가 동시 계측.
- **데이터 수집**: 제어 PC가 입력(리더각)·출력(팔로워 Present Position)을 **동일 클럭으로 20Hz 동시 기록** (앱 [검사 기록] 버튼 → `inspection_logs/*.csv`). 원시 틱 값 병기(추적성).
- **분석·판정**: `inspection_report.py` 가 CSV에서 KPI를 산출하고 `inspection_criteria.json` 기준으로 등급 판정, 마크다운 보고서 자동 생성.
- **측정계 검교정**: 분석기는 합성 데이터 자기검증(`--selftest`)을 내장 — 지연/게인/오프셋/드리프트/반복/정지시간을 알고 주입한 뒤 복원 정확도를 확인하고, 불량 장치(게인 0.5, 무동작)와 계측 이상(음의 지연), 미측정 KPI 검출 같은 거짓 합격/불합격 경로도 검사한다(26항목). **보고서 생성 시 자동 실행되어 머리말에 결과가 기록**되며, 실패 시 해당 보고서는 신뢰 불가로 표기된다.

## 3. KPI 정의

모든 KPI는 분석기가 동일 정의로 산출한다(코드 = 정의의 구현, KPI 번호는 코드 주석과 1:1).

| # | KPI | 정의 | 단위 |
|---|---|---|---|
| K1 | 지연 (latency) | 입력→출력 속도 신호의 정규화 교차상관 최대 지연(포물선 보간). 상관 피크 <0.5면 측정불가 처리 | ms |
| K2 | 추종 오차 (tracking) | K1 지연 보상 후 \|출력−입력\|, 상수 정렬오프셋(중앙값) 제거 → RMS / P95 / 최대 | ° |
| K3 | 스케일 충실도 (gain) | 출력 ≈ a·입력 + b 최소제곱의 a (1.0=충실). 판정값은 \|a−1\| | — |
| K4 | 정지 안정성 (wander) | 입력 정지(<1.5°/s, ≥2s, 정착 0.5s 제외) 중 출력 위치 피크-피크 최대값 | ° |
| K5 | 드리프트 (drift) | 정지 구간별 오차 중앙값의 시간 추세(≥3구간, 시간폭 ≥60s 필요) | °/min |
| K6 | 반복 정밀도 (repeatability) | 기준 포즈(±2.5°) 재방문(≥3회) 시 출력 자세 산포(max−min). ISO 9283 포즈 반복정밀도 개념 차용 | ° |
| K7 | 안전 정지 (safety stop) | **라벨에 'cut'이 포함된 마크**(입력원 차단 순간) → 출력 전 관절 속도 <2°/s 가 0.5s 지속될 때까지의 시간 + 그 사이 잔여 이동량. 마크별 개별 판정, 최악 등급 반영. 'cut' 없는 마크는 주석으로만 기록 | s, ° |

**판정 전처리 규칙**: ①첫 마크 이후 구간은 K1~K6에서 제외(입력-출력 대응이 깨지므로; K7은 전체 사용) ②입력 가동폭 5° 미만 관절은 비검사 ③각 채널 **원시(보간 전) 유효표본** 80% 미만이면 해당 채널 판정 제외+경고 ④시나리오가 요구하는 KPI(criteria의 scenario_kpis)가 미측정이면 종합판정을 '판정불가'로 강등 — 미측정이 합격처럼 보이는 것 차단 ⑤cut 마크 후 기록이 2.5s 미만이면 해당 K7은 판정불가(F 아님).

## 4. 합격 기준 (등급)

기준값은 `inspection_criteria.json` (판정기와 단일 출처). **자체 선언 사양이며 외부 표준 인증이 아님**을 명시한다. 근거: 등급 B는 본 장치의 용도(일반 텔레옵 → OpenArm 전환)에 필요한 최소 성능으로 설정 — 지연은 텔레옵 사용성 문헌의 통용 범위(100~300ms에서 성능 저하 시작)를, 정확도·반복은 기존 실측 베이스라인(j1 RMS 3.09°, 어깨 0.9~2.5°, 손목 9.5~20.7°)에 퇴행 마진을 더해 설정.

| KPI | A (정밀) | B (일반·**전환 게이트**) | C (데모) | 비고 |
|---|---|---|---|---|
| K1 지연 | ≤120ms | ≤180ms | ≤300ms | |
| K2 RMS — 리더 모드 | ≤2° | ≤4° | ≤8° | |
| K2 RMS — IMU 어깨·팔꿈치(j1~4) | ≤3° | ≤6° | ≤12° | |
| K2 RMS — IMU 손목 j6 / j5·j7 | ≤6°/≤8° | ≤12°/≤15° | ≤20°/≤25° | 롤 채널 구조 한계 반영 |
| K2 P95 | RMS 기준의 ~2.5배 | 〃 | 〃 | criteria.json 참조 |
| K3 \|gain−1\| | ≤0.05 | ≤0.10 | ≤0.20 | |
| K4 배회 | ≤0.5° | ≤1.5° | ≤3° | |
| K5 드리프트 — 리더 | ≤0.2°/min | ≤0.5 | ≤1.0 | |
| K5 드리프트 — IMU(j5 제외 / j5) | ≤0.5 / ≤2 | ≤2 / ≤10 | ≤5 / ≤30 | j5=yaw 직결 채널 |
| K6 반복 — 리더 / IMU | ≤1° / ≤3° | ≤2° / ≤5° | ≤4° / ≤8° | |
| K7 안전 정지 | ≤0.8s | ≤1.2s | ≤2.0s | 정지 실패=F |

**종합 판정**: 시나리오 내 산출된 전 KPI의 최악 등급. **OpenArm 전환 게이트 = 해당 시나리오 전부 B 이상**(단 j5 드리프트는 C 허용 + 운용 대책 문서화 조건부).

## 5. 검사 시나리오 (절차)

각 시나리오는 앱의 [검사 기록] 시작→동작→중지 1회 = CSV 1개. 시나리오 ID를 입력칸에 기입(예: `S1_track_j1_leader`). 공통 전제: 모터 연결+토크 ON+전달 중, 기준기(리더) 연결.

| ID | 명칭 | 절차 | 산출 KPI |
|---|---|---|---|
| S1 | 단일관절 추종 | 관절별로 가동범위 70% 진폭 왕복 ≥60s (저속→중속→빠르게 혼합) | K1,K2,K3 |
| S2 | 결합동작 추종 | 다관절 동시 자유 동작 ≥120s, 중간에 정지 ≥3회(각 ≥3s) 포함 | K1~K5 |
| S3 | 정지 안정성 | 임의 자세에서 입력 완전 정지 60s × 2자세 | K4 |
| S4 | 드리프트 | [30s 동작 + 8s 정지] 반복 ×10분 | K5 |
| S5 | 반복 정밀도 | 시작 자세에서 출발→임의 동작→**시작 자세 복귀 후 3s 정지**, 10회 반복 | K6 |
| S6 | 안전 정지 | 동작 중 입력원 차단(ESP 전원/리더 분리)과 **동시에 [마크]** 클릭, 3회 | K7 |
| S7 | 가동범위 충실도 | 각 관절을 한계까지 천천히 왕복 | K3 + 포화 관찰 |
| S8 | 연속 운용 | 일반 사용 30분 연속 기록 | 무고장 + read_fail + K5 |

각 시나리오는 **리더 모드와 IMU 지그 모드 각각** 수행한다(모드는 CSV 메타에 자동 기록되어 판정 기준이 자동 분기). 사람 착용 IMU 검사는 기준기가 없으므로 본 기준서의 합부 대상이 아니며(분석기가 '참고용'으로 표시), 지그 결과 + 별도 착용 평가(drift_ab_test)로 보완한다.

**절차 주의사항**:
- 기록 시작은 **[전달 시작] 이후** (출력 0점 베이스가 기록 첫 틱에 고정됨). 기록 중 [기준 자세 재설정]을 누르면 자동으로 `base_changed` 마크가 찍히고 그 이후 데이터는 추종 KPI에서 제외되므로, 원칙은 **재기록**.
- IMU 지그 검사 시: 리더 연결이 전달 모드를 'leader'로 자동 전환하므로, 기록 전에 전달 모드 콤보가 **'보정+분해(calib_ik)'인지 확인** (잘못되면 모드 메타 오기록 → 잘못된 기준으로 판정).
- S6: 차단과 **동시에** [마크] 클릭, 라벨에 'cut' 포함(예: `S6_cut`), 마크 후 최소 3초 기록 유지.

## 6. 측정 불확도와 한계 (보고서 명기 사항)

- 지연(K1) 불확도: 리샘플+포물선 보간 ±15ms(selftest 실측 ±13ms) + **리더 폴링 버퍼 지연 평균 ~10-20ms 과소측정**(입력은 60Hz 폴링 버퍼, 출력은 즉석 읽기) → 합산 **±35ms**로 본다. K1은 과소측정 방향이므로 경계값 근처(B 180ms)에서는 보수적으로 해석. 음(−)의 지연이 측정되면 계측 이상으로 자동 판정불가 처리.
- 각도 분해능: 리더·팔로워 엔코더 0.088°/틱. K2 A기준(2°) 대비 충분.
- K7의 마크 타이밍: 수동 클릭 → **사람 반응 ±0.3s**. 차단과 동시 클릭 절차 준수, 3회 실시해 최악값 반영(판정기 동작)·중앙값 병기 권장.
- 관측자 효과: 검사 읽기(20Hz GroupSyncRead)는 릴레이와 같은 스레드/버스를 쓰므로 **모터 무응답 시** 타임아웃(~35ms)이 릴레이 틱을 지연시킬 수 있음. 정상 통신 시 영향은 틱당 ~2-3ms 수준. 장애 재현 시나리오에서는 이 영향을 보고서에 명기.
- 기준기 한계: 리더암 자체의 백래시/링크 출렁임은 기준 오차에 포함됨(기존 지그 검증에서 지적된 사항) — 빠른 동작 시나리오에서 수 ° 수준 가능.
- 입력·출력 0점은 기록 시작 시점 자세 — 절대 정렬 차이는 K2에서 '정렬 오프셋'으로 분리 보고(판정 제외).
