# 밴드형 리드디바이스 ↔ 시뮬레이션 텔레오퍼레이션 — 연동 검증 보고서

> 작성일 2026-06-15 · 대상: 밴드형 리드디바이스(IMU 3밴드) → **시뮬레이션 OpenArm** 실시간 텔레옵 연동
> 목적: **실물 OpenArm 투입 전에**, 사람이 IMU 밴드를 끼고 움직이면 시뮬 로봇이 그대로 따라오는지를 시뮬레이션으로 검증한다.
> (※ 미니 팔로우암 실물 텔레옵의 통신·안전·정확도는 별도 `종합검증보고서_BandLead텔레옵_2026-06-15.md` 참조)

## 0. 요약

밴드형 리드디바이스(팔에 착용한 IMU 3밴드)의 동작을 **실물 제어 앱과 동일한 calib_ik 분해기**로 7관절각으로 풀어, 시뮬레이션 OpenArm(viz·three.js, MuJoCo)에 실시간으로 흘려보내 추종시켰다. 시뮬레이션이기 때문에 **실물을 위험에 두지 않고** 부호·매핑·작업공간·자기충돌·중력낙하를 미리 전부 확인할 수 있었다.

| 항목 | 결과 |
|---|---|
| 실시간 텔레옵 연동 | IMU 밴드 → bridge(calib_ik) → viz OpenArm **실시간 추종 성립** (데모영상) |
| 분해기 동일성 | 실물 앱과 **같은 calib_ik + 속도클램프/점프디바운스** 사용 → 시뮬 결과가 실물에 그대로 이전 |
| 기구학 검증(SV1~7) | 부호·매핑 확정, 손목 작업공간 303% 초과·자기충돌 54% 정량화 |
| 물리 검증(MuJoCo) | torque-off=낙하(최대 183°), 위치홀드/홈복귀=낙하 0° 실증 |

**결론:** 밴드 리드디바이스로 시뮬 OpenArm을 실시간 텔레옵하는 연동이 성립하며, 실물 전환에 필요한 부호/매핑/작업공간/안전정책을 시뮬에서 모두 확정했다.

## 1. 연동 구조 — 밴드에서 시뮬까지

```
[밴드형 리드디바이스]                  [PC]                          [시뮬레이션]
 IMU 밴드 ×3 (어깨/상완/전완)   ┌─ bridge.py ───────────────┐
 WT901C 9축                     │  UDP 4210 수신              │
        │ RS485 Modbus          │  calib_ik 분해 → 7관절각    │   SSE    ┌──────────────┐
        ▼                       │  smooth_joints              │ ───────▶ │ viz (three.js)│
   [ESP32] ──UDP(4210)────────▶ │   · 속도클램프 180°/s        │  8000    │ OpenArm URDF  │
                                │   · 점프디바운스 45°·3프레임  │          │ 실시간 추종    │
                                └────────────────────────────┘          └──────────────┘
                                            │ (물리 검증은 별도)
                                            ▼
                                   [MuJoCo OpenArm MJCF, 8kg, 중력]
```

### 1.1 핵심 설계 — "시뮬과 실물이 같은 코드"

bridge.py(`viz/bridge.py`)는 **viz의 구버전 JS IK가 아니라 실물 제어 앱과 동일한 `calib_ik` 분해기**를 import해서 사용한다(`bridge.py:31`). 따라서:

- 분해 알고리즘(멀티스타트 + 브랜치 연속성 + FK 잔차 게이트 <2°)이 실물과 동일
- 안전 평활화도 실물 등가 — `smooth_joints()`(`bridge.py:53`):
  - **속도 클램프** `MAX_VEL_DEG_S = 180°/s` (esp 펌웨어와 동일 상수)
  - **점프 디바운스** `JUMP_DEG = 45°`, `JUMP_CONFIRM = 3프레임` — 한 프레임 튐은 무시, 3프레임 연속이면 진짜 동작으로 추종

→ **시뮬에서 본 거동이 그대로 실물 거동**이라는 점이 이 연동의 가치. 부호·매핑·클램프가 시뮬↔실물 일관.

### 1.2 매핑·부호 (viz에서 확정 → OpenArm 적용)

- `userSign = {1:-1, 2:1, 3:1, 4:-1, 5:1, 6:-1, 7:1}` — j1·j4·j6 방향 반전
- `jointMap = {…, 6:7, 7:6}` — calib_ik 출력 j6↔j7을 OpenArm 관절에 스왑 매핑
- viz와 caring_openarm이 **둘 다 enactic openarm_description 포크**라 이 부호/매핑이 실물에 그대로 유효

## 3. 기구학 검증 (viz, SV1~7) — 시뮬에서 미리 잡은 것들

viz(three.js + OpenArm URDF)는 관절을 명령각에 두는 **기구학** 시뮬 → 매핑·부호·작업공간·연속성·교차오염을 사람 동작 입력으로 검증. 기록은 `viz/sim_logs/*.csv`.

| 시나리오 | 검증 | 결과 | 기록 |
|---|---|---|---|
| **SV1 순수동작/부호** | 단축 동작 → 단축 관절, 부호 방향 | 매핑(j6↔j7 스왑)·부호(j1·j4·j6 반전) 확정 | `SV1_pure_motions.csv` 외 |
| **SV2 교차오염** | 한 축 동작이 타축으로 누출 | **보정축 합성입력 0% 누출**(시스템 무결). 사람 동작의 누출분은 동작 부정확이 원인 | `SV2_crosstalk.csv` |
| **SV3 작업공간** | OpenArm 관절한계 초과 | **손목 303% 초과** → 펜스/스케일 정책 필요 | `SV3_workspace.csv`, `SV3_workspace.svg` |
| **SV6 자기충돌** | 팔 모으는 자세 충돌 | 기구학으론 관통 미검출 → MuJoCo로 이관(§4.3) | `SV6_self_collision.csv`, `.svg` |

→ SV2(교차오염)는 **시뮬에서 합성 입력으로 0% 누출을 입증**해, 사람 측정에서 보이던 누출이 "시스템 결함"이 아니라 "사람 동작의 부정확함"임을 분리했다. 시뮬 검증의 전형적 가치.

## 4. 물리 검증 (MuJoCo) — 기구학으로 못 보는 것

viz는 강체를 명령 위치에 고정 → 중력·접촉·관성을 못 본다. MuJoCo(OpenArm MJCF, 8.0kg, 중력 −9.8)로 물리를 검증.

### 4.1 torque-off = 낙하 (실물 안전정책의 핵심 발견)

토크 OFF 3초:

| 자세 | 최대 관절낙하 | 손끝 추락 |
|---|---|---|
| 중립(팔 아래) | 28° | 0.9cm |
| 팔 앞 수평 | 90° | 62cm |
| **팔 들어올림** | **183°** | **81cm** |

→ 미니암(torque-off=멈춤)과 **정반대**. OpenArm은 토크 끊으면 추락.

### 4.2 위치홀드 / 홈복귀 = 낙하 0° (사용자 제안 실증)

| 정책 | 결과 |
|---|---|
| **위치 홀드**(텔레옵 끊김 시 freeze) | 편차 0.00~0.06°, 외란에도 복귀 |
| **홈 복귀 셧다운**(정상 종료) | 홈 오차 0.00°, 40°/s 제어 하강 |

→ 사용자 제안("값이 끊기면 이전 관절값으로 대기")이 물리적으로 옳음. **위치제어가 토크를 유지하면 중력은 자동으로 버틴다.**

### 4.3 자기충돌 (SV6 자세 → MuJoCo 접촉)

팔을 몸통 쪽으로 모으는 자세에서 **261/480 샘플(54%) 충돌** — 어깨(right_link1↔link3), 팔↔베이스. 사람의 자연스러운 동작이 OpenArm에선 자기충돌 → **실물 자기충돌 펜스 필수.**

## 6. 결론 & 실물 OpenArm 전환

1. **밴드 리드디바이스 ↔ 시뮬 OpenArm 실시간 텔레옵 연동 성립** — 데모영상으로 시연. 사람 팔 동작 → calib_ik 분해 → 시뮬 로봇 동기 추종.
2. **시뮬과 실물이 같은 분해기/클램프**를 쓰므로, 시뮬에서 확정한 부호(j1·j4·j6 반전)·매핑(j6↔j7 스왑)·작업공간(손목 303% 초과)·자기충돌(54%)·낙하정책이 그대로 실물에 이전된다.
3. **실물 전환 시 새로 만들 것**은 `IMU UDP → calib_ik → 7관절각 → ROS 토픽` 어댑터 + IMU 신선도 stale-freeze 래퍼뿐. caring_openarm에 E-stop·중력보상·속도클램프·충돌회피·MuJoCo 통합은 이미 구현돼 있다. (세부 [[caring-openarm-integration]])
4. ⚠ **안전정책 주의:** 미니암의 "이상 시 torque-off"는 OpenArm에 옮기면 낙하 → "위치 홀드 / 홈 복귀 / 브레이크"로 재설계.
