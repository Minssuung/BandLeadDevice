# 미니 팔로우암 테스트 플랜 — OpenArm 전환 게이트 (2026-06-12, FC 2종 검증 반영 최종판)

> 목적: 실물 OpenArm(7DoF, ROS2)으로 가기 전에 **미니 팔로우암(DYNAMIXEL 7축)에서 검증 가능한 모든 것**을 합격점과 함께 확정한다.
> 원칙: 미니암 통과 = 센싱→분해→안전로직 **설계**의 검증. **모터 매핑/부호/한계/동역학 + 전송계층 안전장치 구현은 OpenArm에서 0%부터 재검증.**

## A. 회귀 — 재플래시 검증

| # | 테스트 | 방법 | 합격점 |
|---|---|---|---|
| A1 | 수신율 | imu_gap_probe.py 60초 + **ESP USB 시리얼 STATUS**(PC 도구는 qo 필드 미표시) | rate ≥ 60Hz(목표 ~90Hz), 시리얼서 qo_ok 증가·qo_fail≈0, poll_fail=0, csum_fail=0 |
| A2 | 도착 균일성(정지) | imu_gap_probe.py 60초 | p50 ≈ 1/rate ±20%, **>80ms 끊김이벤트 ≤5건/분**(프로브 요약치 그대로), DTIM 버스트 판정 없음 |
| A2b | 도착 균일성(**동작 중**) | 팔 휘젓기 60초 중 gap probe (기존 실측은 전부 정지 상태 — 몸=RF 차폐물) | A2와 동일 |
| A3 | 오일러 스케일 정합 | 센서 0°/45°/90° 기울여 표시값 대조 | ±5° 이내. ※quat-온리 동작 중 오일러=역산값이라 수정 경로(폴백) 미실행 — 폴백 검증은 코드리뷰로 갈음 |
| A4 | IK 경로 무회귀 | 기준설정 후 정지, ik로그 | 전 관절 \|j\|<2°, res(어깨 imu1/손목 imu3)<2°, NaN 0건 |
| A5 | rpy(비-IK) 모드 | 오일러 차분 경로 = 스케일 수정 직격(감도 1/1.82) | 작동 확인만 — IK가 주력, 게이트 아님 |

## D. 안전 — 결함 주입 (OpenArm 전 최중요)

의도적 고장 주입. **각 항목 시작 전 전원 스위치 손 닿는 곳 확인.**

| # | 결함 주입 | 합격점 |
|---|---|---|
| D1 | 전달 중 ESP 전원 차단 (3센서 동시 stale) | **0.7s 내 freeze** — 차단~freeze 사이 이동은 마지막 유효 목표 방향 보간뿐, 새 방향 운동 0 ("즉시 정지" 아님, stale 문턱만큼 지연 정상) |
| **D1a** | **단일 IMU stale** (imu2 RS485 선 1가닥 분리 또는 펌웨어 강제 skip) — **실제 발생 이력 있음**(imu2 25% stale 사건). 패킷은 계속 오므로 stale 게이트 미작동 → 틀린 자세로 계속 움직이는, freeze보다 위험한 모드 | PC가 센서별 신선도를 구분해 freeze/경고하는지 확인. **구분 못 하면(예상됨) → 센서별 신선도 필드 구현이 OpenArm GO 조건** |
| D1b | **WiFi만 끊김**(AP 차단/PC 와이파이 토글 — ESP는 융합 지속, yaw 리셋 없음 = D1과 다른 복구 경로) | freeze→복구 시 램프인, 점프 0 |
| D1c | freeze 문턱 아래 열화: `tc netem`으로 300~600ms 지연 버스트 | 보간+속도클램프가 흡수, 경고 외 이상거동 0 |
| D2 | D1 상태에서 팔 자세 옮기고 ESP 복구 | 신선도 게이트+램프인(15°/s×1s). **램프 종료 후 거동 포함 판정**: 목표 괴리 >30°면 풀속(180°/s) 추종이 시작되는지, yaw 리셋 경고(60° 점프 감지)가 뜨는지 확인 — 풀속 추종 확인되면 "큰 괴리 시 전달 자동정지" 보강 검토 |
| D3 | 전달 중 토크 OFF→팔 크게 이동→토크 ON | 재시드+램프인, 풀속 추격 0 |
| D4 | 전달 중 기준설정 | 모터베이스 재캡처(구현 확인됨), 런지 0 |
| D5 | 관절한계 초과 자세 | 한계에서 멈춤+**포화 로그(W2)** 출력, 복귀 정상, 발산 0 |
| D6 | 팔꿈치 충돌 방향 끝까지 | ID4 **800틱**(코드 주석의 850은 오기) EEPROM 벽, 프레임 충돌 0 |
| D7 | 빠른 휘두름(>180°/s 상당) | 동기화 보간으로 경로모양 유지(관절별 쳐짐 없음), 속도클램프 경고만 |
| D8 | **m5 90° 베이스 어긋남** | **게이트 교체(FC2 치명)**: ①W1 정합성 게이트 구현+동작 확인(불일치 시 시작 거부) ②재현 시도는 한 세션 10연속이 아니라 **조건 분산 30회** — 앱 콜드스타트 직후×10, ESP 리부팅 직후×5, 토크 토글 직후×5, 장시간 세션 후×5, D테스트 사이 인터리브×5. (10회 무재현은 95% CI로 발생률 <26%밖에 못 보장 — 4번에 1번꼴 결함도 통과하는 수준. 슬루는 속도만 줄이지 방향을 못 고침) |
| D9 | 앱 정상 종료(창 닫기) | closeEvent→전 모터(1~8) TORQUE_DISABLE (코드 확인됨) |
| D9b | 앱 **강제 종료**(`kill -9`) | closeEvent 미호출 → 모터가 마지막 Goal에 토크 ON 유지될 것 — **거동을 문서화**(OpenArm 하트비트/명령만료 요구사항 도출용. 미니암 합격점은 '문서화 완료') |
| D10 | 드리프트 워치독 | j5 천천히 흐르는 상황에서 경고 출력(30s 스로틀) |
| D11 | 전달 중 **모터 버스 분리/복구**(USB 또는 모터 1개 케이블) | 크래시 없이 전달 정지, 재연결 시 베이스 재캡처 강제 |
| D12 | 앱 동결 2초(`kill -STOP`→`-CONT`) | 재개 시 위치 슬루+속도클램프 작동, 점프 0 |
| (선택) | 2중 결함: stale 중 기준설정, 토크 토글 중 stale | 크래시/런지 0 |

## F. OpenArm 전환 게이트 (종합 판정)

**GO 조건 (전부 충족)**:
1. A1~A4, A2b, B1~B5, C1~C4, D1~D12, E1 합격 + B6/D9b 문서화 완료
2. m5 90°: 원인 확정+수정 **또는** W1 정합성 게이트 검증 + 조건분산 30회 무재현
3. D1a 결과에 따라: 센서별 신선도 구분 미구현이면 → **구현 후 GO**
4. **전송계층 의존 안전장치 재검증 계획** (FC2 치명 — D 합격은 DYNAMIXEL 계층 코드의 합격임): 위치 슬루/freeze/램프인/재시드/충돌벽/TORQUE_DISABLE/Profile Velocity를 ROS2 계층에 재구현 → **OpenArm 드라이런+저속에서 D-시리즈 전체 재실행**
5. **실패-안전 정책 재설계** (FC2 치명 — 미니암에서 검증 불가): 미니암의 정답 "이상 시 torque off"는 OpenArm에선 **중력 낙하** = 오답. 자세별 torque-off vs 유지(브레이크/중력보상) 정책 + e-stop 시 낙하 경로 분석
6. OpenArm 측 준비물: 물리 e-stop, **명령-실위치 괴리 워치독**(8차 FC 권고 미반영분 — 충돌 감지 proxy, 단계 프로토콜 4 전 구현), **ROS2 데드맨/명령 만료**(DDS는 발행자 죽어도 마지막 명령 latch 가능 — "PC 죽으면 멈춘다"가 기본 성립 안 함), **자기충돌·작업공간 펜스**(미니암 충돌벽=EEPROM 1개와 다른 새 클래스 — 몸통/테이블/사람이 모두 도달 반경 안), **j5 롤 운용 정책**(27°/min → OpenArm 그리퍼가 2분에 50°+ 틀어짐: 재기준 주기 강제 또는 채널 제한), 단계 프로토콜 0~5 문서화
7. OpenArm에서 종단지연 재측정 (C4는 전송계층 교체로 무효)

**미니암 결과를 신뢰하면 안 되는 것 (OpenArm 0% 재검증)**:
- IK_SIGN 전부 (드라이런으로 재측정), JOINT_LIMIT_DEG/충돌 한계 (기구 실측)
- 속도/가속 프로파일 (150/80은 미니암 기준 — OpenArm은 보수값에서 재시작)
- 전송 계층 전체 (DYNAMIXEL SDK→ROS2) + 그 위의 안전장치 구현 (조건 4)
