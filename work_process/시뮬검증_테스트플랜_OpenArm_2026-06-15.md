# 시뮬(viz OpenArm) 운동학 검증 — 테스트 플랜 & 프레임워크 (2026-06-15)

## 0. 목적과 범위

밴드형 리드디바이스 → **OpenArm URDF** 텔레옵을 **실물 위험 0**으로 선검증.
실물 esp 앱과 **동일한 분해기(calib_ik.py)**를 bridge.py가 구동하므로, 여기서 맞춘 매핑·부호가 그대로 실물 이식 가능.

| 검증함 (운동학) | 검증 안 함 (실물/ROS2 영역) |
|---|---|
| 매핑(어느 동작→어느 관절) | 안전장치(freeze/워치독 — DYNAMIXEL/ROS2 종속) |
| 부호/방향 | 동역학(중력·관성·토크) |
| 작업공간 (사람 vs OpenArm 한계) | 실패-안전(OpenArm torque-off=중력낙하) |
| 교차오염 / 연속성 / FK잔차 | 통신지연·전류·발열 |

## 1. 프레임워크 구성

| 도구 | 역할 |
|---|---|
| `viz/bridge.py` | UDP 수신 → calib_ik 분해 → 3D 렌더 + **CSV 로깅**(IMU자세·관절각·FK잔차) |
| `viz/index.html` | OpenArm URDF 실시간 3D + [기준 설정]·[부호토글]·**[시뮬 기록]·[마크]** |
| `viz/sim_report.py` | CSV → **작업공간·교차오염·연속성·FK잔차** 분석 + **SVG 시계열** + md 보고서 |
| 화면 녹화 | 3D 동작 **영상**(아래 8절) |

## 2. 사전 준비 (매 세션)

1. 메인 앱(esp_imu_pc_control3.py) **종료** (UDP 4210 충돌)
2. `cd ~/dev_ws/BandLeadDevice/viz && python3 bridge.py` → 브라우저 `http://localhost:8000`
3. IMU 밴드 착용, 전원 인가 후 2~5초 정지(자이로 자동캘)
4. 상태바에 수신율(Hz) + "bridge 분해(calib_ik = 실물 동일)" 확인
5. 중립(차렷) 자세 → **[기준 설정]**

## 3. 테스트 시나리오

각 시나리오: [시뮬 기록] 시작 → 동작 시작에 [마크](라벨) → 동작 → 끝에 [마크] → 다음 동작… → [기록 중지]. 마크 라벨 쌍이 sim_report의 분석 구간이 됨.

| ID | 동작 | 마크 라벨 | 검증 항목 | 합격 기준 |
|---|---|---|---|---|
| **SV1 매핑·부호** | 순수 동작 7종 각각 (어깨 굴곡/벌림/비틀, 팔꿈치, 손목 롤/위아래/좌우) 천천히 1회 | 각 동작명 (예 `shoulder_flex`) | 의도 관절만 움직이나 / 방향 맞나 | 의도 관절이 최대 진폭, 부호 정방향 |
| **SV2 교차오염** | 결합 동작 (어깨+팔꿈치 동시, 손목 3축 동시) | `combo_shoulder_elbow` 등 | 순수 동작 분리도 | **누수율 < 15%** (sim_report) |
| **SV3 작업공간** | 각 관절 **사람 최대 가동**까지 (특히 손목 위아래/좌우 끝까지) | `ws_wrist` 등 | OpenArm 한계 초과 여부 | 초과 관절 식별 (손목 ±45° 예상 초과) |
| **SV4 연속성** | 빠른 휘두름 + 극단 자세(손목 꺾기, 어깨 풀회전) | `fast_swing` | 점프/발산 백스톱 | **프레임 점프 < 30°**, FK잔차 폭주 0 |
| **SV5 반복성** | 중립→특정 자세→중립 ×5 (같은 목표 자세) | `repeat` | 같은 자세 재현 | 복귀 시 관절각 산포 (시계열로 육안) |
| **SV6 자기충돌** | 팔을 몸통/반대팔 쪽으로 (도달 범위 극단) | `self_collision` | URDF 링크 충돌 | **3D 영상 육안** (링크 관통 여부) |

| **SV7 작업공간 펜스** | 손목 등 한계 초과 동작 + [펜스 ON] | 클램프·경고 작동 | 한계에서 멈춤 + ⚠ 표시, [펜스 OFF] 시 초과 통과(대조) |

> 권장: SV1·SV3는 필수(매핑·작업공간 확정), SV2·SV4는 품질, SV5·SV6·SV7은 심화.

## 3-A. viz로 할 수 있는 안전 vs MuJoCo/실물로 넘길 것

| 안전 항목 | viz(기구학) | MuJoCo(물리) | 실물 |
|---|---|---|---|
| 작업공간 펜스(한계 클램프) | ✅ **SV7 (구현됨)** | — | 확인 |
| 분해 발산/점프 백스톱 | ✅ SV4 | ✅ | 재실측 |
| **중력 낙하 / 실패-안전 정책** | ❌(강체 고정) | ✅ **토크OFF 자세별 낙하** | 확인 |
| 접촉/자기충돌 힘 | △ 거리만(SV6 육안) | ✅ 접촉력 | 확인 |
| 물리 e-stop·전류·발열 | ❌ | ❌ | 실물만 |

→ **viz 단계 = 운동학(SV1~6) + 작업공간 펜스(SV7)까지.** 중력 낙하·접촉력은 MuJoCo, e-stop·전류는 실물.

## 6. 현재까지 확정값 (viz 화면 토글 — localStorage 영구저장됨)

| 항목 | 값 | 검증 수준 |
|---|---|---|
| 부호 `userSign` | **{1:−1, 2:+1, 3:+1, 4:−1, 5:+1, 6:−1, 7:+1}** | j1·j4·j6 반대 육안 확인 / j2·j3·j5·j7=+1은 SV1 정밀검증 필요 |
| 손목 매핑 `jointMap` | **j6↔j7 스왑** (OpenArm 손목 축 정의가 미니암과 반대) | 좌우입력→위아래출력 확인 후 스왑 |
| 작업공간 주의 | 손목 j6·j7 = OpenArm ±45° (사람 손목 > 이 범위 → 초과 예상) | SV3로 수치화 |

> 이 값은 viz 기본값+localStorage에 저장됨(새로고침/재실행 유지). j2·j3·j5·j7 부호는 "눈으로 안 틀려 보임" 수준이라 SV1 순수동작 측정으로 확정 필요.

## 7. 미니암 IK_SIGN과의 관계

- 미니암(DYNAMIXEL) IK_SIGN = {1:−1, 2:−1, 3:1, 4:1, 5:−1, 6:1, 7:−1}
- OpenArm은 **회전축 정의가 달라** 부호·매핑이 다름 → SV1으로 OpenArm 전용 값 확정
- 확정된 `userSign`/`jointMap` = OpenArm 이식 시 출발점 (ROS2 컨트롤러에 반영)
