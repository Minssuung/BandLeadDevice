# 밴드형 리드디바이스 ↔ 미니 팔로우암 텔레오퍼레이션 테스트 결과 (2026-06-12)

## 1. 테스트 대상·목적

- **리드 디바이스**: 팔 착용 IMU 밴드 3개 (WT901C-485, 윗팔/아래팔/손) → ESP32(RS485 Modbus 폴링) → WiFi UDP
- **팔로워**: DYNAMIXEL 7축 미니 팔로우암 (PC 앱 esp_imu_pc_control3.py — 보정+IK 분해 → GroupSyncWrite)
- **목적**: 사람이 밴드를 착용하고 움직일 때 팔로우암이 **빠르고·정확하고·안전하게** 따라오는지 실측 검증

## 2. 통신·지연 성능 — ✅ 우수

| 항목 | 측정값 | 비고 |
|---|---|---|
| IMU 수신율 | **54Hz** (쿼터니언-온리 폴링) | 어제 30.8Hz → 개선 |
| 동작 중 수신 품질 | **57.8Hz, 끊김(>80ms) 0건/분** | 팔 휘저으며 60초 실측 — 몸 RF 차폐에도 안정 |
| 종단 지연 (사람→로봇) | **~113ms** (88~125ms) | 폰 슬로모 240fps 상호상관 실측 |
| 출력단(목표→모터) | 지연 ~150ms, 게인 0.96~0.99 | 모터가 목표를 충실 추종 |

당일 개선 사항: PC 와이파이 절전 OFF(끊김 24배↓), IMU 내부필터 98Hz, 릴레이 100Hz + One Euro 필터, 쿼터니언-온리 폴링, 오일러 1.82배 스케일 버그 수정.

## 3. 안전 검증 — ✅ 결함 주입 실측 통과 (핵심 성과)

실물에서 일부러 고장을 내서 안전장치가 실제로 막는지 확인:

| 결함 주입 | 기대 동작 | 결과 |
|---|---|---|
| IMU 선 전체 분리 (버스 죽음) | 0.7s 내 freeze | ✅ `IMU 끊김 — freeze` |
| **한 IMU만 분리** (패킷은 정상 수신 지속) | 0.4s 내 freeze | ✅ `🛑 imu3 센서 단독 stale — freeze` |
| 선 재연결 (센서 재부팅, yaw 리셋) | 경고 + 저속 복귀 | ✅ `64° 급변 경고` + 점프 디바운스 + 램프인 15°/s |
| 베이스 정합성 (전달 시작 시) | 읽기 오류 시 시작 거부 | ✅ 정합 검사 실행 확인 (`정합 OK` 로그) |

특기: **단일 IMU 사망은 STATUS상 정상처럼 보이는**(avail=1) 가장 교묘한 위험 모드인데, 신규 age 필드 기반 감지가 정확히 잡음. "틀린 자세로 계속 추종"하는 최위험 시나리오 차단 입증.

추가 안전장치(당일 구현): 명령-실위치 괴리 워치독(0.5s 순회, 지속 30°+ 괴리 시 freeze), 전달 시작 램프인, 토크 재인에이블 재시드.

## 4. 정확도·안정성 측정 (사람 착용, 블랙박스 KPI)

검사 프레임워크(앱 [검사 기록] + inspection_report.py, 자기검증 34/34)로 3회 측정.

### 측정 결과: 종합 **C** (B 문턱 근접)

| 채널 | 배회(°) | 드리프트(°/min) | 등급 |
|---|---|---|---|
| j1/j2/j6/j7 | 0.5~1.2 | 0.05~0.85 | **B** |
| j3 어깨비틀 | 1.44 | **−2.35** (2회 재현) | C |
| j4 팔꿈치 | 1.64 | −0.04 | C |
| j5 손목롤 | 2.14 | 2.60 | C |

### ⭐원인 규명 — 센서는 무죄 (당일 핵심 발견)

"드리프트가 진짜 센서 탓인가, 정지 자세가 매번 달랐던 것 아닌가"라는 의문 검증을 위해 **IMU를 책상에 5분 정지(사람 없음=자세 변수 0) × 2회** 측정:

- **정지 센서 yaw 드리프트 ≈ 0.00°/min (3개 전부)** — 9축 자기치유 우수
- 1회차 imu3에서 일회성 자기교란 홉(−5.6°) 1건 — 2회차 재현 안 됨

→ **텔레옵 측정의 드리프트는 100% 비-센서 원인** (정지 자세 재현 오차 또는 빠른 동작의 yaw 킥). 센서 보정(필터링) 불필요. **C의 병목은 하드웨어 한계가 아니라 동작 속도·측정 방법.**

### B 확정 시도 (추가 4회 측정) — "유일한 벽 = j3 어깨비틀" 규명

정지 시 팔 고정 + 첫 정지구간 안정화 조건으로 4회 추가 측정. 결과 패턴이 명확해짐:

**최종(가장 깨끗, 정지 5구간·118초) 측정 — 6/7 관절 B 이상:**

| 채널 | 배회(°) | 드리프트(°/min) | 등급 |
|---|---|---|---|
| j1 | 0.09 | 0.18 | **A** |
| j2 | 1.10 | 0.25 | B |
| **j3 어깨비틀** | **1.76** | **−2.83** | **C** ← 유일한 벽 |
| j4 | 0.53 | −0.92 | B |
| j5 손목롤 | 1.31 | −1.39 | **B** |
| j6 | 1.06 | −1.22 | B |
| j7 | 0.68 | −1.15 | B |

- **j5 손목롤 해결**: 이전 C의 원인은 "첫 정지구간 잔여 진동"(한 구간만 10°, 나머지 0.18·0.00). 첫 구간 안정화하니 B. → 측정 프로토콜 문제였음(시스템 성능 아님).
- **j3 어깨비틀이 유일한 벽**: 배회(1.76)·드리프트(−2.83) 둘 다 C. 어깨비틀은 사람이 의식적으로 고정/재현하기 가장 어려운 자유도라 정지 중 흔들리고 매 정지마다 자세가 미묘하게 달라짐.
- **j3 드리프트 4회: −2.44 / −2.35 / +3.81 / −2.83** — 3회가 −2.4~−2.8 음수로 일관. 정지 센서는 0이므로 "사람 착용+동작 환경에서 윗팔 heading의 일관된 표류" 가능성 → 일관 방향이면 운용(재기준)·보정으로 잡을 여지(미검증). +3.81은 그 세션 이상치.

**B를 못 넘는 건 j3 단일 채널.** 진짜 B 인증은 GT(지그) 필요(FC: 사람 착용=반증 전용). 사람 착용으로는 **"j3 어깨비틀 1채널만 빼면 전 관절 B"**가 정직한 천장.

## 5. 종합 판정

| 영역 | 판정 |
|---|---|
| 통신·지연 | **A급** (113ms, 끊김 0) |
| 안전 | **통과** (결함 주입 실측, 다층 안전장치 작동) |
| 정확도·안정성 | 측정 **C** — 단 **6/7 관절 B 이상(j1 A)**, 유일한 벽 = j3 어깨비틀(사람 착용 자세 재현 한계) |

**밴드형 리드디바이스 ↔ 미니 팔로우암 텔레옵 = "통신 A급 + 안전 실증 + 정확도 6/7 관절 B"** 상태. 종합 C는 j3 어깨비틀 단일 채널 때문이며, 이는 시스템 성능이 아니라 사람 착용 측정의 구조적 한계(어깨 회전 자유도 재현 불가). 전 관절 B 인증은 지그(GT) 측정 필요.
